论在计算机模拟下实现概率计算

写在前面

我在数学书上看了用频率算概率这节,突发奇想,想用我为数不多的C++知识来实现一下(一年没碰C++了)

正文

1.计算硬币的正反面概率

(1)使用rand来进行随机数计算

使用标准库 cstdlib 的随机数函数进行随机生成数后求余,进行10000次随机数求余计算
主体代码如下

#include <iostream>
#include <cstdlib>
using namespace std;
int main() {
	int a;
	double target = 0;
	double total;
	for (int i = 0; i < 10000; i++) {
		a = rand() % 2;
		cout << a << '\t';
		if (a != 0) {
			target++;
		}
		total++;
	}
	double p;
	p = target / total;
	cout << "\n目标事件" << target << '\n' << "事件总数" << total << '\n' << "p=" << p;
}

输出结果如下

(前面随机数生成不显示)
目标事件5035
事件总数10000
p=0.5035

不过我发现这个程序的随机数生成的p(概率)值都一样(大草)

(2)通过time函数进行随机数计算

使用标准库 ctime 的随机数函数进行随机生成数后求余,进行10000次随机数求余计算
主体代码如下

#include <iostream>
#include <ctime>
using namespace std;

int main() {
	int a;
	double target = 0;
	double total;
	for (int i = 0; i < 10000; i++) {
		a = time(0) % 2;
		cout << a << '\t';
		if (a != 0) {
			target++;
		}
		total++;
	}
	double p;
	p = target / total;
	cout << "\n目标事件" << target << '\n' << "事件总数" << total << '\n' << "p=" << p;
}

(懒)
输出结果如下

(随机数生成省略)
目标事件5252
事件总数10000
p=0.5252

(这玩意全都是0,1一块来的草)
